# OpenCode Plugin Development - Repository Guide

This is the `create-opencode-plugin` scaffolding tool repository. It generates new OpenCode plugin projects with a single command.

## Useful Commands

### Development

```bash
# Run the scaffolding tool in development mode
bun dev

# Build the scaffolding tool for distribution
bun build

# Run all checks (format, lint, typecheck)
bun check

# Format code with oxfmt
bun fmt

# Check formatting without modifying files
bun fmt:check

# Lint source code with oxlint
bun lint

# Auto-fix lint issues
bun lint:fix

# Typecheck TypeScript without emitting
bun typecheck

# Publish to npm (runs prebuild which runs check + build)
npm publish
```

### Testing the Scaffolded Output

```bash
# Create a test plugin (uses MOCK_PROMPTS=true for CI)
MOCK_PROMPTS=true bun dev

# Then test the generated plugin
cd my-opencode-plugin
bun install
bun dev
```

## Best Practices and Guidelines

### Code Style

- **Formatter**: oxfmt (not Prettier)
  - Tabs for indentation
  - Single quotes for strings
  - No semicolons
  - Trailing commas everywhere
  - 100 character print width
  - LF line endings
- **Linter**: oxlint
  - No unused variables (error)
  - Correctness/suspicious/perf rules as warnings
  - Import plugin enabled
  - TypeScript plugin enabled

### TypeScript Configuration

- Target: ES2022
- Module: ESNext with bundler resolution
- Strict mode enabled
- No emit (use Bun for bundling)

### Git Conventions

- Commit messages should follow conventional commits format
- The `.gitignore` excludes: `node_modules/`, `dist/`, `.env`, `tmp`, `.test`, `*.log`, `.cline`

### Plugin Template Conventions

When modifying the plugin template (`template/` directory):

- Use placeholder variables with `{{VARIABLE_NAME}}` syntax
- Available placeholders:
  - `{{PACKAGE_NAME}}` - npm package name
  - `{{PLUGIN_NAME}}` - PascalCase plugin class name
  - `{{OPENCODE_PLUGIN_VERSION}}` - Latest @opencode-ai/plugin version
  - `{{OPENCODE_SDK_VERSION}}` - Latest @opencode-ai/sdk version
  - `{{DESCRIPTION}}` - Plugin description
  - `{{AUTHOR}}` - Author name

### Plugin Development Guidelines

For plugins generated by this tool:

- **Runtime**: Always use Bun (>=1.3.0), not npm/yarn/pnpm
- **Logging**: Use `ctx.client.app.log()` for structured logging, not `console.log`
- **Shell Commands**: Use Bun shell (`ctx.$`) for executing system commands
- **Type Safety**: Always type the plugin context parameter
- **Hook Return**: Return an object with only the hooks you need

### Environment Variables

- `MOCK_PROMPTS=true` - Skip interactive prompts (for CI/testing)
- `OPENCODE_CONFIG_CONTENT` - Used by `dev.ts` to pass plugin config to OpenCode

### Windows/PowerShell Compatibility

- Use `bun` instead of `npm`/`yarn`/`pnpm`
- Use PowerShell-compatible commands when scripting
- Paths use backslashes but Bun accepts forward slashes
- Use `Remove-Item -Recurse` instead of `rm -rf`
- Use `Get-Content` instead of `cat`
- Use `write` tool for files with special characters

### Security Considerations

- Never commit `.env` files
- The scaffolding tool fetches package versions from npm registry
- Plugin permissions are controlled via the `permission.ask` hook
- Be cautious with `tool.execute.before` hooks that block tool execution

### Performance

- oxlint and oxfmt are significantly faster than ESLint/Prettier
- Bun's bundler is used for building (faster than tsc)
- Template copying uses synchronous filesystem operations (acceptable for small file sets)
- Package version fetching uses `Promise.all` for parallel requests

### Common Patterns

#### Custom Tool Registration

```typescript
import { tool } from '@opencode-ai/plugin'

return {
	tool: {
		myTool: tool({
			description: 'Tool description',
			args: {
				input: tool.schema.string(),
			},
			async execute(args, context) {
				return `Result: ${args.input}`
			},
		}),
	},
}
```

#### Blocking Dangerous Operations

```typescript
'tool.execute.before': async (input, output) => {
  if (input.tool === 'read' && output.args.filePath.includes('.env')) {
    throw new Error('Cannot read .env files')
  }
}
```

#### Auto-Approve Safe Operations

```typescript
'permission.ask': async (input, output) => {
  if (input.tool === 'read') {
    output.status = 'allow'
  }
}
```

### Troubleshooting

- **Type errors**: Run `bun typecheck` to verify types
- **Formatting issues**: Run `bun fmt` to auto-format
- **Lint errors**: Run `bun lint:fix` to auto-fix
- **Build fails**: Run `bun check` to run all checks first
- **Template not updating**: Ensure `dist/` is rebuilt with `bun build`

### Publishing Workflow

1. Update version in `package.json`
2. Run `bun check` to verify everything passes
3. Run `npm publish`
4. The `prepublishOnly` script will automatically build
5. Users can then install with `bun create opencode-plugin my-plugin`
